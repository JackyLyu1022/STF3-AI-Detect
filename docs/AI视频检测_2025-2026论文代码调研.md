## 1. 对你现有参考文献的替换建议

你原来的参考主要是 CNNDetection 2020、UniversalFakeDetect 2023、DIRE 2023、NPR 2024、FreqNet 2024。建议替换成下表：

| 原参考 | 建议替换为 | 替换理由 | 对项目实现的落点 |
|---|---|---|---|
| CNNDetection / UniversalFakeDetect | **ReStraV (NeurIPS 2025)** 或 **UNITE (CVPR 2025)** | 不再只讨论单帧 CNN/CLIP，而强调视频表征轨迹或全帧视频检测 | `SpatialBranch` 可换成 DINOv2/SigLIP 特征抽取 + 轻量分类头 |
| NPR | **D3 (ICCV 2025)** | NPR 是图像局部像素关系；D3 是视频二阶时序差分，更贴近 AI 视频检测 | `TemporalBranch` 从 `abs(frame[t]-frame[t-1])` 升级到二阶中心差分/特征差分 |
| FreqNet | **WaveRep (NeurIPS 2025)** + **SPAI (CVPR 2025)** | 2025 频域/小波/谱学习方法更适合支撑频域分支 | `FrequencyBranch` 改成 FFT + wavelet band + spectral reconstruction / augmentation |
| DIRE | **STALL (CVPR 2026)** 或 **D3 (ICCV 2025)** | DIRE 推理成本高；STALL/D3 是训练少或免训练，更适合课程项目 | 加一个 training-free score 分支或作为 OOD baseline |

## 2. 主推论文卡片

### 2.1 D3: Training-Free AI-Generated Video Detection Using Second-Order Features

- **会议**：ICCV 2025
- **CCF 参考**：A
- **论文**：https://arxiv.org/abs/2508.00701
- **CVF**：https://www.openaccess.thecvf.com/content/ICCV2025/papers/Zheng_D3_Training-Free_AI-Generated_Video_Detection_Using_Second-Order_Features_ICCV_2025_paper.pdf
- **代码**：https://github.com/Zig-HS/D3
- **核心思想**：使用二阶中心差分特征捕获 AI 视频在运动/动态上的不自然性，属于 training-free 检测。
- **和本项目的匹配度**：极高。
  - 你现在的 `TemporalBranch` 是一阶相邻帧差：`abs(frames[:,1:] - frames[:,:-1])`。
  - D3 可以直接升级为二阶差分：`x[t+1] - 2*x[t] + x[t-1]`，也可以在 XCLIP/DINO/ResNet 特征空间做二阶差分。
  - D3 README 明确包含 GenVideo 数据集组织和推理流程，对你的 GenVideo-Val 最友好。
- **建议实现方式**：
  1. 新建 `src/models/d3_temporal_branch.py`。
  2. 对采样帧或 frame embedding 做二阶差分。
  3. 输出统计量：mean/std/max/min/energy，或者接一个 SmallCNN/MLP。
  4. 与原 STF3 融合，命名 `stf3_d3`。

### 2.2 ReStraV: AI-Generated Video Detection via Perceptual Straightening

- **会议**：NeurIPS 2025
- **CCF 参考**：A
- **论文/会议信息**：https://nips.cc/virtual/2025/poster/118520
- **代码**：https://github.com/ChristianInterno/ReStraV
- **核心思想**：用 DINOv2 提取逐帧 embedding，把视频看成表征空间轨迹；真实视频轨迹更“直”，生成视频在曲率、转角、步长统计上异常。
- **和本项目的匹配度**：很高。
  - 论文代码只用 21 维几何特征 + MLP，适合 RTX 4060 和课程项目。
  - 可以不训练大模型，只抽 DINOv2 特征再训练轻量分类器。
- **建议实现方式**：
  1. 新增 `RepresentationGeometryBranch`。
  2. 采样 16 或 24 帧，用 DINOv2 ViT-S/14 抽特征。
  3. 计算 step distances、turning angles、曲率统计。
  4. 与 `Spatial/Frequency/Temporal` 拼接，形成 `stf3_restrav`。

### 2.3 Seeing What Matters / WaveRep

- **会议**：NeurIPS 2025
- **CCF 参考**：A
- **项目页**：https://grip-unina.github.io/WaveRep-SyntheticVideoDetection/
- **代码**：https://github.com/grip-unina/WaveRep-SyntheticVideoDetection
- **核心思想**：不让检测器只看语义缺陷，而通过小波分解和频带替换增强，让模型关注生成器引入的低层 forensic traces。
- **和本项目的匹配度**：极高，尤其是频域分支。
  - 你的 `FrequencyBranch` 当前是 FFT log-magnitude + SmallCNN。
  - WaveRep 可作为数据增强：对训练帧做 wavelet band replacement，提升跨生成器泛化。
- **建议实现方式**：
  1. 先不改模型，只在 dataset transform 中加入小波频带增强。
  2. 再把 `FrequencyBranch` 从 FFT 单路改成 `FFT + Wavelet` 双路。
  3. OOD split 上重点比较：`stf3` vs `stf3 + WaveRepAug`。

### 2.4 SPAI: Any-Resolution AI-Generated Image Detection by Spectral Learning

- **会议**：CVPR 2025
- **CCF 参考**：A
- **项目页**：https://mever-team.github.io/spai/
- **代码**：https://github.com/mever-team/spai
- **核心思想**：只用真实图像进行 masked spectral learning，学习真实图像频谱分布；生成图像作为 OOD，通过 spectral reconstruction similarity 检测。
- **和本项目的匹配度**：中高。
  - 它是图像检测，但视频可以逐帧跑，再做时间平均/最大池化。
  - 可以替代你现在参考的 FreqNet，作为 2025 CVPR 频域理论支撑。
- **建议实现方式**：
  1. 先离线抽帧，对每帧计算 SPAI score。
  2. 视频级聚合：mean/max/top-k mean。
  3. 与 STF3 logits 做 late fusion。

### 2.5 NSG-VD: Physics-Driven Spatiotemporal Modeling for AI-Generated Video Detection

- **会议**：NeurIPS 2025 Spotlight
- **CCF 参考**：A
- **论文**：https://openreview.net/forum?id=HiBoJLCyEo
- **代码**：https://github.com/ZSHsh98/NSG-VD
- **核心思想**：基于概率流守恒提出 Normalized Spatiotemporal Gradient，衡量视频时空动态是否符合真实世界物理过程，并用 MMD 做检测。
- **和本项目的匹配度**：中等偏高，但实现成本较高。
  - 优点：论文叙事强，非常适合写“算法提升”。
  - 难点：涉及预训练扩散模型/梯度估计，复现成本高。
- **建议实现方式**：
  - 短期不建议完整复现；可以抽取其“物理一致性/时空梯度统计”思想，做简化版 `NSG-lite`：空间梯度幅值 / 时间变化幅值 的统计特征。

### 2.6 STALL: Training-free Detection of Generated Videos via Spatial-Temporal Likelihoods

- **会议**：CVPR 2026
- **CCF 参考**：A
- **项目页**：https://omerbenhayun.github.io/stall-video/
- **代码页**：https://github.com/OmerBenHayun/stall-video
- **论文**：https://arxiv.org/abs/2603.15026
- **核心思想**：用真实视频校准集白化 embedding，分别建模帧级空间似然和帧间转移时序似然，融合得到视频真实性分数。
- **和本项目的匹配度**：高，尤其适合作为 OOD baseline。
- **注意**：目前 GitHub 看起来更偏项目页，真正可运行代码完整度需要下载复核。
- **建议实现方式**：
  1. 用训练集中 real 视频作为 calibration set。
  2. 用 DINOv2/DINOv3/SigLIP 抽帧特征。
  3. 实现白化 + Gaussian log-likelihood + percentile fusion。
  4. 不训练 classifier，直接在 OOD 测试集汇报 AUC。

## 4. 建议最终项目算法路线

### 路线 A：最稳、最适合课程项目

> **STF³-D3-WaveRep**

- Spatial：保留 ResNet18 或换 DINOv2 frozen feature。
- Temporal：引入 D3 二阶差分分支。
- Frequency：保留 FFT，同时加入 WaveRep 小波增强。
- Fusion：原 MLP head 拼接。
- 实验：Random split + OOD split；重点看 OOD AUC/F1。

优点：实现成本可控、论文全是 2025 CCF-A、和现有代码改动最小。

### 路线 B：更有新意但仍轻量

> **STF³-ReStraV**

- 在现有三分支外增加 ReStraV 21-D 表征轨迹几何特征。
- 不需要大规模训练，只用 DINOv2 frozen embedding。
- 论文叙事：从像素/频谱伪影扩展到“视频表征流形几何异常”。

### 路线 C：冲高质量论文叙事

> **Real-calibrated Training-free OOD Detector**

- 参考 STALL：只用 real calibration set，建模 spatial/temporal likelihood。
- 用作 OOD detector 或和 STF3 late fusion。
- 论文亮点：无需见过 fake generator，强调跨生成器泛化。

## 6. 我建议你下一步先做的实现顺序

1. **D3-lite TemporalBranch**：成本最低，直接替换当前帧差分支。
2. **WaveRep augmentation**：不改模型先加增强，跑 OOD 对比。
3. **ReStraV feature branch**：抽 DINOv2 21-D 几何特征，做 late fusion。
4. **SPAI/STALL 作为外部分数**：先离线算 score，再和 STF3 logits 融合。

这样你的论文方法就能从“传统三分支融合”升级为：

> **基于二阶时序动态、频域小波伪影增强与表征轨迹几何的 AI 生成视频检测方法**

这个表述比原来的 2020-2024 参考更新，而且 2025/2026 顶会支撑更充分。
