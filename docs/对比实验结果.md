# STF3 外部对比实验记录（Comparison Log）

> 文件名沿用当前项目中的 `comparision.md`。  
> 最后更新：2026-06-15。  
> 正类定义：`fake / AI-generated = 1`。  
> 主对比表统一采用 `precision_0.95_recall_max` 作为验证集阈值选择策略：在 validation set 上保证 Precision ≥ 0.95 的前提下最大化 Recall，然后固定该阈值到 OOD test。

---

## 1. 统一实验协议

### 1.1 数据划分

| Split | 路径 | 样本数 | 用途 |
|---|---|---:|---|
| OOD train | `data/GenVideo-Val/splits/ood_train.csv` | 12,273 | 可训练模型的训练集 |
| OOD val | `data/GenVideo-Val/splits/ood_val.csv` | 2,429 | 阈值选择 / 模型选择 |
| OOD test | `data/GenVideo-Val/splits/ood_test.csv` | 3,598 | 最终测试 |

说明：

- STF3 和 TALL 使用 OOD train / val / test。
- D3 是 training-free baseline，不使用 OOD train，只在 val 上选分数方向与阈值，再测试 test。
- WaveRep 使用官方预训练权重，不在本项目数据上重新训练，只在 val/test 上推理与阈值校准。
- D3、WaveRep、TALL 在 test 上均有 13 个视频读取/打分失败，因此有效 test N 为 3,585；STF3 主结果使用完整 test N=3,598。

### 1.2 为什么主表统一用 `precision_0.95_recall_max`

本项目目标是 AI 视频检测系统。相比单纯最大化 Accuracy 或 F1，系统部署时更需要控制误报，同时尽量减少 fake 漏检。因此主表统一采用：

```text
precision_0.95_recall_max:
在 validation set 上筛选 Precision >= 0.95 的阈值，
再选择 Recall 最高的阈值，
最后固定到 test set。
```

这样可以避免“为每个模型挑 test 上最好看的指标”，使不同模型的 operating point 更公平。

## 3. D3 对比实验

### 3.1 如何进行对比

D3 被作为 external training-free baseline。实验没有重新训练 D3，而是复用其核心 D3_model 与 second-order dynamics scoring 机制，在相同的 GenVideo-Val OOD val/test 上重新计算分数。

流程：

```text
ood_val.csv  -> D3 score -> 在 val 上确定 score orientation 和 threshold
ood_test.csv -> D3 score -> 固定 val threshold 后测试
```

### 3.2 参数设置

|---|---|

| OOD val valid / errors | 2,429 / 0 |
| OOD test valid / errors | 3,585 / 13 |

### 3.3 完整结果

| Objective | Threshold | Test ACC | Test AUC | Test AP | Test F1 | Precision | Recall | Balanced ACC | TN | FP | FN | TP |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|---:|---:|---:|

D3 在 OOD test 上 AUC=0.9513，说明 second-order temporal degradation cue 具有较强排序能力。但在统一的高 precision operating point 下，Recall 只有 0.6288，FN 达到 774，说明 D3 的阈值化检测在当前 OOD 生成器上较保守，容易漏检 fake 视频。

## 6. 主对比表：STF3 vs D3 / WaveRep / TALL

> 主表只保留每个模型的一个最终 operating point：`precision_0.95_recall_max`。  
> 粗体表示该列最优值；`FN/FP` 越低越好，其余指标越高越好。  
> 按当前汇报要求，主表删除 ACC、AP、Balanced ACC，仅保留 AUC、F1、Precision、Recall、FN 和 FP。

| 类型 | Method | Test N | AUC | F1 | Precision | Recall | FN ↓ | FP ↓ | 定位 |
|---|---|---:|---:|---:|---:|---:|---:|---:|---|
| Proposed | `STF3-New R7_224_FAKEW12` | 3,598 | 0.9848 | **0.9499** | 0.9740 | **0.9271** | **153** | 52 | 主模型；三分支可解释融合 |
| External training-free | `D3 XCLIP-16 L2` | 3,585 | 0.9513 | 0.7662 | 0.9806 | 0.6288 | 774 | 26 | 二阶动态退化线索 |
| External pretrained | `WaveRep DINOv2 G4` | 3,585 | 0.9573 | 0.9207 | 0.9758 | 0.8715 | 268 | 45 | 官方预训练取证检测器 |
| External trainable | `TALL-SWIN 3ep best.pt` | 3,585 | **0.9898** | 0.9296 | **0.9929** | 0.8739 | 253 | **23** | 强 Swin thumbnail-layout baseline |

### 6.1 对比解读

1. **TALL 是当前最强外部可训练 baseline。**  
   在主表保留的指标中，TALL 的 AUC、Precision 和 FP 控制最优，说明强 Swin thumbnail-layout 时空建模对当前 OOD 数据非常有效，尤其误报较少。

2. **STF3 仍然具有竞争力。**  
   STF3 的 F1、Recall 和 FN 控制最好，说明在统一高 precision operating point 下，它在整体 F1 与 fake 视频召回能力上仍然最强，漏检 fake 更少。同时，三分支结构提供了更明确的 spatial / temporal / frequency 证据解释。

3. **D3 有较强排序能力，但阈值化检测较弱。**  
   D3 的 AUC=0.9513，说明动态退化线索有效；但在高 precision operating point 下 Recall 明显下降，FN 达到 774。

4. **WaveRep 比 D3 更强，但仍偏保守。**  
   WaveRep 的 Precision 较高，但 Recall 和 FN 不如 STF3/TALL，说明官方预训练取证检测器在当前数据上会漏掉一部分 fake。

5. **不能简单表述为 STF3 全面超过所有外部模型。**  
   更严谨的表述是：STF3 作为可解释三分支融合框架，在 OOD 检测中表现强且稳定，尤其在 F1、Recall 和 FN 控制上具有优势；TALL 作为强外部可训练 Swin baseline，在 AUC、Precision 和 FP 控制上更强。

## 7. 答辩推荐表述

### 7.1 对 D3 的表述

> D3 is evaluated as a training-free external baseline. We do not quote the original paper results directly; instead, we run D3 on the same GenVideo-Val OOD validation/test split and select the threshold only on validation. D3 achieves a high AUC, indicating that second-order dynamics are informative, but its thresholded recall is much lower than STF3.

中文：

> D3 被作为 training-free 外部 baseline。我们没有直接引用原论文结果，而是在相同 GenVideo-Val OOD validation/test split 上重新计算 D3 分数，并只在 validation set 上选择阈值。D3 的 AUC 较高，说明二阶动态线索有效，但固定阈值后的 Recall 明显低于 STF3。

### 7.2 对 WaveRep 的表述

> WaveRep is evaluated using the official pretrained G4 checkpoint without retraining on GenVideo-Val. Frame-level logits are averaged into video-level scores, and the threshold is selected on validation. WaveRep is conservative with high precision, but it misses more fake videos than STF3.

中文：

> WaveRep 使用官方 G4 预训练权重进行评估，不在 GenVideo-Val 上重新训练。我们将帧级 logit 平均为视频级分数，并在 validation set 上选择阈值。WaveRep 的 Precision 较高，说明较保守，但相比 STF3 漏检更多 fake 视频。

### 7.3 对 TALL 的表述

> TALL-SWIN is evaluated as a strong external trainable spatiotemporal baseline under the same OOD train/validation/test protocol. Its three-epoch best checkpoint slightly outperforms STF3 in F1 and balanced accuracy, mainly due to stronger precision and fewer false positives. This is reasonable because TALL uses a powerful Swin Transformer backbone and thumbnail-layout multi-frame modeling. However, STF3 remains competitive and has slightly fewer false negatives, while providing a more interpretable spatial-temporal-frequency fusion framework.

中文：

> TALL-SWIN 被作为强外部可训练时空 baseline，在与 STF3 相同的 OOD train/validation/test 协议下评估。三轮训练得到的 best checkpoint 在 F1 和 balanced accuracy 上略高于 STF3，主要优势来自更高 Precision 和更少 FP。这是合理的，因为 TALL 使用了更强的 Swin Transformer backbone 和 thumbnail-layout 多帧建模方式。但 STF3 仍然具有竞争力，并且 FN 略少，同时提供了更可解释的 spatial-temporal-frequency 三分支融合框架。

## 8. 总结

本对比实验形成了三类外部参照：

| 对比类型 | 方法 | 作用 |
|---|---|---|
| Training-free temporal baseline | D3 | 验证二阶动态退化线索 |
| Official-pretrained forensic baseline | WaveRep | 验证外部预训练取证检测器 |
| External trainable spatiotemporal baseline | TALL | 验证强 Swin thumbnail-layout 时空模型 |

最终结论：

1. STF3 的三分支融合不是孤立结果；它相对于 D3 和 WaveRep 有明显更好的 Recall/FN 控制。
2. TALL 作为强外部可训练 baseline 略优于 STF3 的部分综合指标，说明强 backbone 和 thumbnail-layout 时空建模在当前数据上非常有效。
3. STF3 的主要贡献应表述为：**可解释、模块化的 spatial-temporal-frequency 三证据融合框架**，而不是声称全面超过所有外部强模型。
4. 如果答辩只展示一个最终主表，建议使用第 6 节表格，并明确所有模型均采用 validation-calibrated `precision_0.95_recall_max` operating point。
